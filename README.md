# Self-Service VLESS Deploy Bot

هر کاربری که با این ربات صحبت کنه، می‌تونه با فرستادن **توکن و Account ID خودش**،
یک نسخه‌ی مستقل از پروکسی VLESS رو روی **اکانت کلودفلر خودش** دیپلوی کنه —
نه اکانت تو.

## مدل امنیتی (مهم، حتماً بخون)

این ربات یه نکته‌ی بنیادی داره که باید برای کاربرات هم شفاف باشه:

- برای دیپلوی خودکار، توکن کاربر **باید** یه لحظه از سرور ربات (که دست توئه) عبور کنه.
- کد این ربات طوری نوشته شده که توکن **در هیچ فایل، دیتابیس، یا لاگی ذخیره نمی‌شه**
  — فقط توی متغیر موقت جاواسکریپت می‌مونه و بعد از دیپلوی از حافظه پاک می‌شه.
- با این‌حال، **از نظر امنیتی، کاربر باید به عملگر این ربات (یعنی تو) اعتماد کنه**
  که کد رو دستکاری نکردی و واقعاً توکن رو جایی لاگ نمی‌کنی. این یه حقیقت
  فنی‌ئه، نه یه محدودیت قابل دور زدن — هر ربات مشابهی همین وضعیت رو داره.
- به همین دلیل توی پیام خوش‌آمد از کاربر خواسته می‌شه فقط یه توکن **محدود**
  بسازه (فقط Workers Scripts + Workers KV Storage)، نه توکن کامل ادمین —
  اگه توکن هرجایی درز کنه، حداکثر آسیب محدود به همون دو تا permission می‌مونه.
- پیام حاوی توکن بلافاصله بعد از دریافت از چت پاک می‌شه.

اگه می‌خوای این ربات رو عمومی/برای غریبه‌ها هم بذاری در دسترس، این نکات رو
براشون هم توضیح بده — شفافیت درباره‌ی این مدل اعتماد بخشی از مسئولیت شماست.

## نصب

```bash
npm install
cp .env.example .env
nano .env   # فقط BOT_TOKEN خودت (توکن این ربات، نه توکن کلودفلر)
npm start
```

## استفاده (از دید کاربر نهایی)

1. `/start` — توضیحات و نکات امنیتی رو می‌بینه
2. `/deploy` — ربات Account ID رو می‌پرسه
3. کاربر Account ID رو می‌فرسته
4. ربات توکن رو می‌پرسه
5. کاربر توکن رو می‌فرسته → ربات فوراً پیام رو پاک می‌کنه و دیپلوی رو شروع می‌کنه
6. در پایان، لینک پنل ادمین (`/admin/<secret>`) و لینک ساب (`/sub/<secret>`) رو می‌گیره

هر کاربر یک `personal-vless-xxxxxx` مجزا روی اکانت خودش می‌گیره — کاملاً
مستقل از بقیه.
